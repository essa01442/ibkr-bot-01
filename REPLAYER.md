# محرك إعادة التشغيل (L1 Replay Engine)

## نظرة عامة
محرك `replayer` هو أداة لاختبار التراجعات (Regression Testing) وتهدف إلى ضمان الحتمية (Determinism) في اتخاذ القرارات من قبل `TapeEngine`.
يقوم المحرك بقراءة بيانات السوق التاريخية (Tick data) من ملف CSV، وتمريرها عبر نفس خط الأنابيب (Pipeline) المستخدم في التداول الحي، ثم تسجيل وحفظ القرارات في ملف نتائج بصيغة JSON يُعرف بـ (Golden File).

## صيغة ملف الإدخال (CSV Format)
يجب أن يحتوي ملف الـ CSV على الترويسة (Header) التالية وأن تلتزم البيانات بالأنواع المحددة:
```csv
ts_src,symbol_id,price,bid,ask,volume,side
```
### تفاصيل الأعمدة:
* `ts_src` (u64): الطابع الزمني للبيانات بالميكروثانية.
* `symbol_id` (u32): المعرف الرقمي للرمز.
* `price` (f64): سعر الصفقة/التحديث.
* `bid` (f64): سعر الطلب.
* `ask` (f64): سعر العرض.
* `volume` (u32): حجم الصفقة.
* `side` (String): اتجاه الصفقة (`B` لـ Bid أو `A` لـ Ask).

*مثال:*
```csv
ts_src,symbol_id,price,bid,ask,volume,side
1700000000000,1,10.0,9.99,10.01,100,B
1700000001000,1,10.05,10.04,10.06,200,A
```

## طريقة الاستخدام

### توليد ملف النتائج الذهبي (Golden File)
لتمرير البيانات وإنشاء ملف النتائج الذهبي:
```bash
cargo run --bin replayer <مسار_ملف_CSV> <مسار_الملف_الذهبي.json> [مسار_إعدادات_TOML_اختياري]
```
*ملاحظة:* إذا لم يتم تمرير ملف إعدادات بشكل صريح، سيستخدم المحرك الافتراضي `configs/default.toml`.

### المقارنة (Comparison)
لمقارنة نتيجتين (مثلاً بين إصدارين من الكود لضمان عدم كسر الحتمية):
```bash
cargo run --bin replayer --compare <الملف_الذهبي_الأول.json> <الملف_الذهبي_الثاني.json>
```
سينتج عن هذا إما `PASS` إذا كانت نسبة التطابق تتجاوز أو تساوي 99%، أو `FAIL` إذا كان هناك اختلاف جوهري في قرارات الدخول/الرفض وتوقيتاتها.
